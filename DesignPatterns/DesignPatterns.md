# 1. Overview

- OO：Object-Oriented，adj. 面向对象的；
  - 封装、抽象、继承、多态； 
- 设计原则：
  - 找出需要被频繁修改的代码，将其与稳定的代码分隔开；
  - 针对接口（或抽象类）编程，而非针对实现编程；
  - 多用组合，少用继承；
    - E.g. 通过组合使用不同的类实现特定行为；



# 2. 策略模式

- 策略模式：
  - 定义算法族，并分别封装，算法之间可相互替换；
  - 策略模式使得算法的变化独立于使用算法的客户；
